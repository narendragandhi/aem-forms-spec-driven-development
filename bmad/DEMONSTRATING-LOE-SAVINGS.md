# How to Demonstrate Level of Effort (LoE) Savings with the AEM Forms BMAD Showcase

This document provides a guide for using this repository to demonstrate and quantify the significant savings in Level of Effort (LoE) achieved through the BMAD methodology, specifically for AEM Forms projects.

## The Core Concept: Manual vs. AI-Agent Workflow

To showcase the value of BMAD, it is essential to contrast it with the traditional AEM Forms development workflow.

### Traditional Baseline (High LoE)
In a typical AEM Forms project, creating custom components is a manual and complex process:
- **Architects** design the component's data binding and authoring dialog requirements.
- **Frontend Developers** must hand-code the React component, ensuring it integrates correctly with the AEM Forms framework. This includes writing the component logic, styling, and the authoring dialog configuration (`dialog.json`).
- **Backend Developers** may need to write Sling Models or services to support the component's data interactions.
This process requires specialized knowledge, involves multiple developers, and is time-consuming.

### BMAD Acceleration (Low LoE)
This repository demonstrates a new, accelerated paradigm:
- **The `bmad/` documentation** (e.g., `03-Architecture-Design/component-design.md`) serves as the **prompts and instructions** for AI agents.
- **The code** in the `ui.frontend.react.forms.af` and `core` modules is the **output generated by these agents**.

The LoE savings come from automating the complex steps of creating custom Adaptive Form components. The effort shifts from manual coding to **prompt engineering and output validation**, a significantly faster and more efficient process.

## How to Demonstrate LoE Savings

Here are two concrete methods to demonstrate the value proposition of the BMAD methodology for AEM Forms.

### 1. Use the "Traceability Matrix" as Proof

The `bmad/traceability-matrix.md` file is a powerful tool for proving the efficiency of the BMAD process.

**How to Use It:**
11.  **Select a Feature:** Pick a forms-related feature from the traceability matrix (e.g., "Loan Application Submission").
12.  **Show the Golden Thread:** Follow the row from left to right to show a direct, unbroken link from a business requirement to a **Form Data Model (FDM)** design, then to a custom component specification, and finally to the implemented code.
13.  **Explain the Value:** This demonstrates a streamlined process that reduces ambiguity and manual hand-offs between business analysts, architects, and developers, directly saving time and reducing errors.

### 2. Conduct a Side-by-Side LoE Comparison for a Custom Form Component

This method provides a quantitative contrast for a common task in AEM Forms projects.

**How to Use It:**
1.  **Choose a Component:** Select the **"Address Lookup"** custom component, as defined in `bmad/02-Model-Definition/content-models.md`. This is a realistic example of a component that requires both frontend and backend logic.
2.  **Estimate Traditional LoE:** In one column, estimate the hours required for a developer to build this manually:
    - Reading the spec and understanding the AEM Forms component framework.
    - Writing the React component (`AddressLookup.js`).
    - Creating the authoring dialog JSON (`dialog.json`).
    - Mapping the component's fields to a Form Data Model.
    - Styling the component to match the theme.
    - Writing unit tests for the component.
    *A reasonable estimate might be 12-16 hours for a developer new to the AEM Forms framework.*
3.  **Present the BMAD Effort:** In a second column, showcase the BMAD workflow:
    - Present the component specification from the "Data Models and Templates" document as the **prompt** given to a "Developer Agent."
    - Show the resulting code files that would be generated in `ui.frontend.react.forms.af` as the **output**.
    - The effort is reduced to prompt engineering, minor adjustments, and validating the generated code.
    *The estimated effort here might be 2-4 hours, showcasing a 4x to 6x reduction in LoE for a single, non-trivial custom component.*

By using these methods, you can clearly and effectively articulate how the BMAD methodology, as demonstrated by this showcase, delivers substantial reductions in cost, effort, and time-to-market for AEM Forms projects.
